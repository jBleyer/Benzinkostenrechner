## Beispiele von Sicherheitsrisiken von Webanwendungen :

1. Phishing – Betrügerische Webseiten oder E-Mails, die Nutzerdaten (z. B. Passwörter) stehlen wollen.
2. Datendiebstahl – Hacker greifen auf persönliche oder vertrauliche Daten zu.
3. SQL-Injection – Angreifer schleusen schädlichen Code in Datenbankabfragen ein, um Daten zu lesen oder zu verändern.
4. Cross-Site-Scripting (XSS) – Schadcode wird in Webseiten eingeschleust, der dann bei anderen Nutzern ausgeführt wird.
5. Session-Hijacking – Angreifer übernehmen eine aktive Benutzersitzung, um sich als der Nutzer auszugeben.
6. DoS (Denial of Service) – Die Webseite wird mit Anfragen überflutet, bis sie abstürzt oder nicht mehr erreichbar ist.

## Maßnahmen zum Schutz von Webanwendungen
      
1. Verschlüsselung (z. B. HTTPS) – Schützt Daten bei der Übertragung vor Abhören.
2. Multifaktor-Authentifizierung (MFA) – Nutzer müssen sich mit mehreren Nachweisen anmelden (z. B. Passwort + Handycode).
3. Sanitizing / Prepared Statements – Säubern von Nutzereingaben, um SQL-Injections und XSS zu verhindern.
4. Sichere Passwörter – Komplexe Passwörter erschweren unbefugten Zugriff.
5. Regelmäßige Updates – Schließen bekannte Sicherheitslücken in Software und Servern.
6. Firewalls & DDoS-Schutz – Filtern schädlichen Datenverkehr und blockieren Angriffe
